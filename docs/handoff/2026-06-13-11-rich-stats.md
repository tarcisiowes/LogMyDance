# 11 — Rich Stats screen

**Date:** 2026-06-13
**Status:** ✅ Done — `tsc --noEmit` clean.

## What was done
- `statsRepo(db).getStats()` aggregates (raw SQL via `db.$client`, parallel queries):
  - total classes, total movements, total practice minutes (`SUM(duration_min)`).
  - this-week / this-month class counts (ISO-date `>=` bounds; week starts Monday).
  - current + longest streak, computed in JS from distinct entry dates (date-fns).
  - breakdowns: by style (LEFT JOIN styles), by mood, by movement status.
- Rebuilt `app/(tabs)/stats.tsx`: overview grid (classes / movements / practice time / current streak with flame), week+month mini-stats, horizontal bar breakdowns (style / mood / status), Beta Diagnostics (app opens + longest streak), and cards linking to Storage & Backup and the new Settings screen.
- Empty-state copy when no data yet. All labels i18n'd (`stats.*`).

## Files touched
- `src/repositories/stats.ts` — **new**. `StatsData` type + `statsRepo`. `type DB = ReturnType<typeof drizzle<typeof schema>>` (NOT `ExpoSQLiteDatabase<>` — that alias doesn't expose `$client`).
- `app/(tabs)/stats.tsx` — rewritten (was a placeholder counting entries/movements/templates).

## Decisions / gotchas
- Streak logic: current streak counts back from today (or yesterday if nothing logged today); longest = max consecutive run over all distinct days.
- Bars use a min 6% width so non-zero counts are always visible; colors from `MOODS` / `MOVEMENT_STATUSES` constants.
- Reload on `useFocusEffect` so stats refresh after logging a class.
