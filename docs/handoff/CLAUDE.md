# docs/handoff/ — work log

One markdown file per completed task. Newest indexed in `README.md`.

Naming: `YYYY-MM-DD-NN-slug.md` (NN = running counter). Each entry records: what was done, files touched, dependencies, decisions/gotchas, manual actions, and how to verify.

## Convention
- Write a handoff doc for every meaningful change and add a row to `README.md`'s index table.
- Note anything not verifiable in-session (native builds, store steps) under "Manual actions".
- Keep them factual + skimmable; they are the project's memory of *why*.
