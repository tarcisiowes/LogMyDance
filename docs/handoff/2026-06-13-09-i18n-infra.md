# 09 — i18n infrastructure (EN + PT-BR)

**Date:** 2026-06-13
**Status:** ✅ Done — `tsc --noEmit` clean. Native rebuild required to test (new native module).
**Sprint:** 4 (Stats + Polish)

## What was done
- Added i18next + react-i18next + expo-localization. Device-locale detection with EN fallback, choice persisted in MMKV (`language` key, already in `preferences`).
- Translation resources for English + Brazilian Portuguese covering: tabs, onboarding, stats, settings, common.
- Type-safe `t()` via `CustomTypeOptions` augmentation — keys are checked at compile time.
- Settings screen with a live language switcher + app version.

## Files touched
- `src/i18n/index.ts` — **new**. i18next init, `detectLanguage()` (saved pref → device `getLocales()` → EN), `changeLanguage()`, `getLanguage()`, `SUPPORTED_LANGUAGES`.
- `src/i18n/locales/en.ts` — **new**. Source of truth; `TranslationSchema = typeof en` (NOT `as const` — would make values literal types and break PT-BR assignment).
- `src/i18n/locales/pt-BR.ts` — **new**. Typed `: TranslationSchema`, so missing/renamed keys fail `tsc`.
- `src/i18n/i18next.d.ts` — **new**. Module augmentation for typed keys.
- `app/_layout.tsx` — `import '@/i18n'` (side-effect init, before any screen). Translated nav title for settings.
- `app/(tabs)/_layout.tsx` — tab titles via `t('tabs.*')`.
- `app/settings.tsx` — **new** (language switcher + version).

## Dependencies
- `expo-localization ~56.0.6` (**native module** — autolinked, needs rebuild)
- `i18next ^26.3.1`, `react-i18next ^17.0.8` (JS only)
- `expo-localization` auto-added to `app.json` `plugins`.

## Decisions / gotchas
- Locale `lng` for PT is `'pt-BR'` (matched from device `languageTag === 'pt-BR'` or `languageCode === 'pt'`).
- `getLocales()` wrapped in try/catch (web has no native module).
- Typed `t()` rejects `string` variables and template-literal keys — pass literal keys, or map explicitly (see `settings.tsx` language label, `onboarding.tsx` `SLIDES as const`).
