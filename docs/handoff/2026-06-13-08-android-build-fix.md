# 08 — Android build fix (namespace + missing icon)

**Date:** 2026-06-13
**Status:** ✅ Fixed — `assembleDebug` BUILD SUCCESSFUL, `app-debug.apk` produced

## Problem
`npx expo run:android` failed at `:app:compileDebugKotlin`:
```
MainActivity.kt: Unresolved reference 'R'
MainActivity.kt / MainApplication.kt: Unresolved reference 'BuildConfig'
```
Then, after a namespace patch, `:app:compileDebugJavaWithJavac` failed:
```
ReactNativeApplicationEntryPoint.java: cannot find symbol com.logmydance.BuildConfig
```

## Root cause
The committed `android/` project was an **inconsistent prebuild**:
- `android/app/build.gradle` → `namespace "com.logmydance"`
- Kotlin sources + `app.json` → `com.logmydance.app`

`R` / `BuildConfig` are generated under the gradle `namespace`, so code in package
`com.logmydance.app` could not resolve them. The manifest's relative `.MainActivity`
refs would also have failed at runtime. This is the classic result of running
`expo prebuild` (non-clean) after the package name changed.

Secondary blocker: `assets/icon.png` (referenced by `app.json`) was never committed —
`assets/` was empty — so `expo prebuild` aborted in the icon mod.

## Fix
1. Generated a placeholder 1024² icon → `assets/icon.png` (script:
   `scripts/gen-placeholder-icon.js`).
2. `npx expo prebuild --clean -p android` — regenerated `android/` from `app.json` as
   the single source of truth. Now consistent: `namespace`/`applicationId` =
   `com.logmydance.app`, matching the Kotlin package and manifest.
3. `:app:assembleDebug` → BUILD SUCCESSFUL (10m, full native compile incl.
   react-native-zip-archive, reanimated/worklets).

## Files touched
- `assets/icon.png` — **new** (placeholder)
- `scripts/gen-placeholder-icon.js` — **new** (regenerates the placeholder)
- `android/**` — fully regenerated by clean prebuild (namespace/applicationId now
  `com.logmydance.app`). An earlier manual 2-line namespace patch to `build.gradle`
  was superseded by the clean prebuild.

## Dependencies
- None new. Clean prebuild relinked the Sprint 3 native modules into `android/`.

## Blockers
- None remaining for the build.

## Manual actions required
1. Replace `assets/icon.png` with real artwork before any store build (it's a flat
   violet square). 1024×1024, no transparency for iOS.
2. Install/run on a device or emulator: `npx expo run:android` (reuses this build —
   fast) or `adb install android/app/build/outputs/apk/debug/app-debug.apk`.
3. Decide whether `android/` stays committed. It's a generated artifact; many Expo
   projects gitignore `android/` + `ios/` and rely on prebuild. If kept, commit the
   regenerated tree so it's consistent for others.
